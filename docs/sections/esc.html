For rare cases, you might need to escape code pieces in CognitiveJS. For example in this documentation code snippets had to be escaped in order to properly display them. In such cases like this, there are different escape definitions for different situations.

First escape method is adding underscore "_" before attributes to make them act like a normal attribute, but tokens inside them will be rendered. This definition is used to mitigate special attribute problem (especially style attribute) which browsers will try to process them before Cognitive does, which would result in unwanted circumstances. Adding an underscore solves that problem. After all the tokens are processed and render is completed, CognitiveJS will remove that underscore then browsers will initiate their own processing.

Second one is element escape definition. Adding cog-escape attribute to the element will completely neutralize the element and its children, so that rendering process will not reach to that element.

    
        <html>
        <head>
            <title>CognitiveJS</title>
        </head>
        <body>

            <p _style="color:{{color}};">Lorem ipsum dolor sit amet.</p>

            <div cog-escape>
                <p cog-prop="style:{color:{{color}}}">{{hello}}</p>
            </div>


            <script src="../dist/cog.js"></script>
            <script>
                cog.data.hello = "Hello World";
                cog.data.color = "#f00";
                cog.render();
            </script>

        </body>
        </html>
